# Deep Research Skill

## 概要

6エージェント構成でトピックを多角的にリサーチし、統合レポートを生成するスキル。

## トリガー条件

以下のキーワードが含まれる場合に発火：
- 「○○について調べて」
- 「○○をリサーチして」
- 「deep research」
- 「詳しく調査して」

## 実行フロー

### Step 0: トピック確認

リサーチトピックが曖昧な場合は以下を確認する：
- リサーチの目的（意思決定のため？学習のため？記事執筆のため？）
- 深さの希望（概要レベル / 実装レベル / 論文レベル）
- アウトプット形式（マークダウン / 箇条書き / 詳細レポート）

### Step 1: Phase 1 - 並列リサーチ（3エージェント同時起動）

Agent ツールを使って以下の3エージェントを**同時に**起動する（並列実行が必須）。

---

**エージェント①: Web調査エージェント**

```
役割: 最新動向・実際の事例・ユースケースの収集

指示:
- WebSearch ツールで「{トピック}」に関する最新情報を検索
- 直近1〜2年の動向・トレンドを重点的に調査
- 実際の導入事例・成功事例・失敗事例を収集
- 公式ドキュメント・主要ブログ・技術記事を参照
- 出力: 最新動向サマリー、事例リスト（各3〜5行の説明付き）
```

---

**エージェント②: 技術深掘りエージェント**

```
役割: 仕組み・背景・原理の深掘り

指示:
- 「{トピック}」の技術的な仕組みを詳細に調査
- なぜその技術/手法が生まれたか（背景・歴史）
- 内部アーキテクチャ・動作原理
- 関連技術・比較対象との違い
- 学術論文・技術仕様書も参照
- 出力: 技術解説（図解できる場合はASCIIアートで）、用語定義リスト
```

---

**エージェント③: 批判的分析エージェント**

```
役割: リスク・限界・反論の収集

指示:
- 「{トピック}」の課題・デメリット・限界を調査
- よくある批判・懐疑的な見方を収集
- 導入コスト・学習コスト・運用コストの現実
- 失敗しやすいパターン・アンチパターン
- 「やってみたけどダメだった」事例も積極的に収集
- 出力: リスクリスト（深刻度付き）、アンチパターン集
```

---

### Step 2: Phase 2 - 統合

3エージェントの出力を受け取り、以下の構造に統合する：

```markdown
## エグゼクティブサマリー
（3〜5行で全体像）

## 最新動向
（エージェント①の結果）

## 技術的詳細
（エージェント②の結果）

## リスク・注意点
（エージェント③の結果）

## 総合評価
（3エージェントの結果を統合した総合判断）
- 推奨度: ★☆☆〜★★★
- 推奨ユースケース
- 避けるべきケース
```

矛盾する情報がある場合は両論を併記し「要確認」とマークする。

### Step 3: Phase 3 - レビュー

以下の品質基準をセルフチェックする：

| チェック項目 | 基準 |
|------------|------|
| 網羅性 | 最新動向・技術詳細・リスクが全て含まれているか |
| 客観性 | 一方的な賛否になっていないか |
| 具体性 | 抽象論だけでなく具体例があるか |
| 矛盾 | 各エージェントの情報に未解決の矛盾がないか |

基準未達の項目がある場合は、該当エージェントを再実行して補完する。

### Step 4: Phase 4 - 最終レポート出力

以下のフォーマットで最終レポートを出力する：

```markdown
# 🔍 Deep Research: {トピック}

**調査日**: {日付}
**調査範囲**: 最新動向 / 技術詳細 / リスク分析

---

## TL;DR
（3行以内の要点）

## 1. 最新動向・事例
...

## 2. 技術的な仕組み
...

## 3. リスク・限界・注意点
...

## 4. 総合評価
- **推奨度**: ★★☆（状況による）
- **こんな人/チームに向いている**: ...
- **こんな場合は避けた方がよい**: ...

## 5. 次のアクション候補
1. ...
2. ...
3. ...

---
*本レポートは Web調査・技術深掘り・批判的分析の3エージェントによる並列リサーチを統合して生成しました。*
```

## 実行時の注意

- **並列実行を必ず守る**: Phase 1 の3エージェントは同時起動すること。順番に実行すると時間が3倍かかる。
- **1エージェント1タスク**: 各エージェントに複数の役割を持たせない。
- **レビューをスキップしない**: 時間短縮のためにStep 3を省略しない。
- トピックが広すぎる場合（例:「AI全般」）は、スコープを絞るよう先に確認する。
